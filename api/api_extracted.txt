DOCUMENT API 
 
CRM sẽ thiết kế link Webhook để nhận API từ PBX ( URL hoặc 
JSON tuỳ vào CRM ) 
 
Dưới đây là các ví dụ theo dạng URL tham khảo : 
 
 
  CUỘC GỌI ĐẾN 
 
Khi có cuộc gọi vào tổng đài sẽ đổ chuông tới máy nhân viên (ví dụ extension: 101): 
 
https://{CRM_URL}/api/callcenter/call_in?phone=0938
363739&queue=999&extension=101&callid=5e052a75-
121b-4c10-bdb4-c4b235b6a677&event=ANSWER 
 
Trong đó: 
 
o  {CRM_URL} : Do CRM cung cấp  
o  phone : Số điện thoại gọi đến 
o  queue : Nhóm trực của số nội bộ ( Nếu Có ) 
o  extension : Số nội bộ của agent 
o  callid : ID của cuộc gọi (mỗi cuộc gọi sẽ có call id khác nhau) 
o  event : Trạng thái cuộc gọi ( RING, ANSWER ) 
 
 
 
  KẾT THÚC CUỘC GỌI  
 
Đây là sự kiện báo cho CRM biết là kết thúc cuộc gọi : 
 
https://{CRM_URL}/api/callcenter/hangup?callid=5e052a75-
121b-4c10-bdb4-c4b235b6a677 
 
Trong đó : 
 
o  {CRM_URL} : do CRM cung cấp 
o  callid : ID của cuộc gọi (mỗi cuộc gọi sẽ có call id khác nhau)

  THÔNG TIN CUỘC GỌI  
 
Khi kết thúc cuộc gọi thì tổng đài có thể trả về thông tin chi tiết cuộc gọi: 
 
https://{CRM_URL}/api/callcenter/calldata?callid=5e052a75-121b-
4c10-bdb4-c4b235b6a677&calldate=2022-10-10 
10:10:10&duration=11 &billsec=7&status=ANSWER&extension=1
01&phone=0938363739&recordingfile=https://ip_tong_dai/link_file
_ghi_am 
 
Trong đó : 
 
o  {CRM_URL} : Do CRM cung cấp 
o  callid : ID của cuộc gọi (mỗi cuộc gọi sẽ có call id khác nhau) 
o  calldate : Thời gian bắt đầu thực hiện cuộc gọi ( định dạng y-m-d h:m:s ) 
o  duration : Thời gian bắt đầu cuộc gọi 
o  billsec : Thời gian bắt đầu thoại 
o  status : Trạng thái cuộc gọi ( ANSWER, BUSY, ...) 
o  extension : Số nội bộ của agent 
o  phone : Số điện thoại khách hàng 
o  recordingfile : Đường dẫn file ghi âm cuộc gọi

  SỰ KIỆN CẬP NHẬT TRẠNG THÁI  CUỘC GỌI 
CỦA AGENT 
 
https://{CRM_URL}/api/Agent/ChangeStatus?extension=101
&status=talk 
 
Trong đó : 
o  {CRM_URL} : Do CRM cung cấp 
o  extension : Số nội bộ của agent 
o  status : Trạng thái của agent (*) 
 
(*) Giá trị trạng thái Agent 
 
Giá trị             Mô tả 
off                 Agent (phone) chưa online 
on                 Agent (phone) sẵn sàng 
pause              Agent (phone) tạm nghỉ 
ring                Agent (phone) đang ring 
talk                Agent (phone) đang talk 
 
  TRẠNG THÁI TRẢ VỀ : 
 
-   Dựa theo HTTP Status Code, Ví dụ : 
 
o  200 : Thành Công 
{“status code” : 200 , “message” : “OK”} 
 
o  404 : Có Lỗi 
{“status code” : 404 , “message” : “Not Found”}

API TỪ  TỔNG ĐÀI   
 
  SỰ KIỆN CLICK2CALL   
 
CRM chạy đường dẫn này method GET kiểu dữ liệu URL :
 
 
https://{PBX_URL}/api/CallControl/dial/from_number/100/
to_number/093xxxxxxx/key/98589afb123863cf4d5cd4ec2c41
072/domain/abc.com 
 
Trong đó :  
 
o  {PBX_URL} : Do tổng đài cung cấp. 
o  api/CallControl/dial : Tên phương thức gọi API của hệ thống. 
o  from_number : Số nội bộ do hệ thống CRM cung cấp. ( vd : 100 ) 
o  to_number : Số khách hàng do hệ thống CRM cung cấp. 
o  key : Khóa của công ty do tổng đài cung cấp. 
o  domain : Tên miền của công ty do tổng đài cung cấp.

  SỰ KIỆN SPY CALL  
 
CRM chạy đường dẫn này method POST kiểu dữ liệu JSON : 
 
https://{PBX_URL}/api/Agent/spy/format/json 
---- 
Data : 
{ 
         “key” : “98589afb123863cf4d5cd4ec2c41072”, 
         “src” : “100”, 
         “dst” : “101”, 
         “mode” : “whisper” 
} 
 
Trong đó : 
 
o  {PBX_URL} : Do tổng đài cung cấp. 
o  api/Agent/spy/format/json : Tên phương thức gọi API của hệ thống. 
o  key : Khóa của công ty tổng đài cung cấp. 
o  src : Extension người thực hiện. 
o  dst : Extension người bị nghe. 
o  mode : 
o  whisper : chỉ nghe. 
o  threeway : nói với cả hai. 
o  caller : nói với người gọi ( caller ) 
o  callee : nói với người nghe
